## Overview

Extract `src/cloud/` (15 files) to `../browse_cloud` and `src/sdk/` (6 files) to `../browse_sdk` as independent repos. `browse_cloud` depends on `@ulpi/browse` as a PUBLISHED npm package via built library output -- not raw `.ts` files. This requires adding a library build target (`dist/lib.mjs`) to browse_cli alongside the existing CLI bundle (`dist/browse.cjs`), plus a types pipeline (`tsc --emitDeclarationOnly`) producing `dist/types/*.d.ts`, plus an `exports` field in `package.json` with `types` conditions. `browse_sdk` has zero CLI deps and extracts cleanly. `src/sdk.ts` (local SDK using BrowserManager directly) stays in `browse_cli`.

Fixes 5 of 7 cloud issues. VM networking and WS streaming for isolated sessions are out of scope.

## Scope Challenge

The critical constraint is the REAL package boundary: `browse_cloud` must import from `@ulpi/browse` as a built npm package, not raw TypeScript source. This means `browse_cli` needs TWO build outputs plus a types pipeline:

1. `dist/browse.cjs` -- the CLI entry point (already exists)
2. `dist/lib.mjs` -- library entry point exporting the ~25 symbols cloud needs
3. `dist/types/lib.d.ts` -- TypeScript declarations generated by `tsc --emitDeclarationOnly`

The `package.json` `exports` field must have `types` conditions pointing to `dist/types/lib.d.ts` and `import` conditions pointing to `dist/lib.mjs`. Without this, `import { SessionManager } from '@ulpi/browse'` won't resolve both runtime AND types in a consumer project.

`tsconfig.json` already has `declaration: true, declarationMap: true, outDir: dist, rootDir: src`. The `build:lib` script must run BOTH esbuild (for `dist/lib.mjs`) AND `tsc --emitDeclarationOnly --declaration --outDir dist/types` (for `.d.ts` files).

Cloud imports from browse_cli (8 import lines across 3 files):

All must be exported from `src/lib.ts` and tests rewritten to import from `@ulpi/browse`. This is a boundary rewrite, not a directory move.

SDK imports nothing from CLI -- the Transport interface is self-contained in `src/sdk/session.ts`.
