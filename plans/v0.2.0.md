# Plan: v0.2.0 — Security, Multi-Instance, Interactions, DX

> Generated: 2026-03-15
> Branch: `feat/v0.2.0`
> Mode: HOLD

## Overview

Ship 10 features that close competitive gaps with agent-browser while preserving our USPs (session multiplexing efficiency, Playwright Locator-backed refs, published benchmarks, simplicity). Features span security (`--content-boundaries`, `--allowed-domains`), multi-instance support (auto-instance servers, per-session folders), DX (`--json`, config file), interactions (dblclick, focus, check, drag, iframe), and distribution (state persistence, platform binaries).

## Scope Challenge

We deeply analyzed agent-browser's implementation (27K lines Rust). Selected 10 features with highest impact-to-effort ratio. Deliberately skipped: action-policy files (over-engineered), `--max-output` (our architecture already solves), cloud providers (not our use case), `find` semantic locators (our ref system is better), auth vault (premature), raw mouse control (rarely needed by AI agents), video recording (requires ffmpeg). Mode is HOLD because we build on existing SessionManager, BrowserManager, and command routing patterns.

## Tasks

### TASK-001: Auto-Instance Servers via PPID

Replace the single shared state file with PPID-based instance isolation. Each parent process (Claude Code instance) gets its own server and Chromium. Sessions within each instance still multiplex on shared Chromium.

In `src/cli.ts`: compute `INSTANCE_ID` from `process.ppid` (or `BROWSE_INSTANCE` env var override). Update `STATE_FILE` to include instance ID: `.browse/browse-server-{instanceId}.json`. Update `LOCK_FILE` similarly. Pass `BROWSE_INSTANCE` to spawned server.

In `src/server.ts`: read `BROWSE_INSTANCE` env var and use it for state file naming instead of `INSTANCE_SUFFIX` (which is port-based). Keep `BROWSE_PORT` as an explicit override.

**Type:** feature
**Effort:** M

**Acceptance Criteria:**
- [ ] Two CLI processes with different PPIDs get separate state files and separate servers
- [ ] Two CLI processes with same PPID (same parent) share one server
- [ ] `BROWSE_INSTANCE=custom` env var overrides PPID-based detection
- [ ] Existing `BROWSE_PORT` behavior unchanged (explicit port still works)

**Agent:** nodejs-cli-senior-engineer

**Priority:** P0

### TASK-004: --content-boundaries (Prompt Injection Defense)

Wrap page-sourced output in CSPRNG nonce-delimited markers so LLMs can distinguish tool output from page content. Only applies to commands that return page-derived content (text, html, snapshot, js, eval, console, links, forms, accessibility).

In `src/cli.ts`: parse `--content-boundaries` flag. Send `X-Browse-Boundaries: 1` header. Support `BROWSE_CONTENT_BOUNDARIES=1` env var.

In `src/server.ts`: detect header. If set, wrap response text for page-content commands in:
```
--- BROWSE_CONTENT nonce=<uuid> origin=<url> ---
<content>
--- END_BROWSE_CONTENT nonce=<uuid> ---
```
Use `crypto.randomUUID()` for nonce (generated once per server process via module-level constant). Commands that don't return page content (status, tabs, screenshot path) are NOT wrapped.

**Type:** feature
**Effort:** S

**Acceptance Criteria:**
- [ ] `browse --content-boundaries text` wraps output in nonce-delimited markers
- [ ] `browse --content-boundaries click @e3` does NOT wrap (action output, not page content)
- [ ] Nonce is a valid UUID, same across all commands in a server session
- [ ] Origin reflects the current page URL

**Agent:** nodejs-cli-senior-engineer

**Priority:** P1

### TASK-005: --allowed-domains (Navigation + Resource Blocking)

Block navigation and sub-resource requests to domains not in the allowlist. Uses Playwright's `context.route()` API for network-level blocking and URL validation before `goto`.

Create `src/domain-filter.ts`: `DomainFilter` class with `isAllowed(url)` method. Support wildcards (`*.example.com` matches `example.com` and subdomains). Case-insensitive.

In `src/cli.ts`: parse `--allowed-domains domain1,domain2` flag. Send `X-Browse-Allowed-Domains` header. Support `BROWSE_ALLOWED_DOMAINS` env var.

In `src/server.ts`: pass allowed domains to `SessionManager.getOrCreate()`.

In `src/session-manager.ts`: when creating a session with allowed domains, call `context.route('**/*', handler)` on the BrowserContext to intercept and block disallowed domains. Also validate URLs in `goto` command before navigation.

In `src/commands/write.ts`: in `goto` case, check URL against domain filter before navigating. Throw clear error if blocked.

**Type:** feature
**Effort:** M

**Acceptance Criteria:**
- [ ] `browse --allowed-domains example.com goto https://evil.com` returns error without navigating
- [ ] `browse --allowed-domains *.example.com goto https://sub.example.com` succeeds
- [ ] Sub-resources (images, scripts, XHR) from disallowed domains are blocked
- [ ] Without `--allowed-domains`, no filtering occurs (backward compatible)

**Agent:** nodejs-cli-senior-engineer

**Priority:** P1

### TASK-009: State Persistence (state save/load)

Add `state save [name]` and `state load [name]` commands. Uses Playwright's `context.storageState()` which captures all cookies + localStorage from all origins in one call (better than agent-browser's single-origin approach).

In `src/commands/meta.ts`: add `state save` and `state load` cases.

`state save [name]`: call `context.storageState()`, write JSON to `.browse/states/{name}.json`. Default name: `default`.

`state load [name]`: read JSON, create new context with `browser.newContext({ storageState: data })`. Replace session's context.

In `src/browser-manager.ts`: add `getContext()` accessor and `replaceContext(storageState)` method that creates a new context with the saved state and migrates tabs.

Register `state` in `META_COMMANDS` in `src/server.ts`. Add `state` to `WRITE_COMMANDS` since `state load` mutates.

**Type:** feature
**Effort:** M

**Acceptance Criteria:**
- [ ] `browse state save mysite` saves cookies + localStorage to `.browse/states/mysite.json`
- [ ] `browse state load mysite` restores cookies + localStorage; subsequent navigation uses them
- [ ] `browse state save` without name defaults to `"default"`
- [ ] `browse state load nonexistent` returns clear error

**Agent:** nodejs-cli-senior-engineer

**Depends on:** TASK-002
**Priority:** P2

## Failure Modes

| Risk | Affected Tasks | Mitigation |
|------|---------------|------------|
| PPID not stable across shells/tmux | TASK-001 | Fall back to `BROWSE_INSTANCE` env var; Claude Code can set it per workspace |
| Playwright route() doesn't catch all sub-resources | TASK-005 | Test with real pages; route('**/*') pattern catches all network requests |
| iframe locator scope breaks ref system | TASK-007 | Snapshot within frame re-scopes refs; document that refs are per-frame |
| storageState() doesn't capture sessionStorage | TASK-009 | Document limitation; sessionStorage is ephemeral by design |
| Bun cross-compile produces broken binary | TASK-010 | Test each binary in CI; fall back to `bun run` if binary fails |
| Config file JSON parse error crashes CLI | TASK-008 | Wrap in try/catch, silently ignore malformed config |
| Content boundary nonce leaked via page JS | TASK-004 | Nonce is per-server-process, not exposed to page; page can't read server memory |
