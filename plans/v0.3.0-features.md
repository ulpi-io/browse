# Plan: v0.3.0 — Headed Mode, Pixel Diff, Clipboard, DevTools Inspect

> Generated: 2026-03-16
> Branch: `feat/v0.3.0`
> Mode: EXPANSION

## Overview

Add 4 features that close competitive gaps and improve debugging: `--headed` flag to show the browser, real pixel-diff image output for screenshot-diff, clipboard read/write commands, and `browse inspect` to open Chrome DevTools on the running page. All leverage existing Playwright APIs — no external dependencies needed.

## Scope Challenge

All 4 features build on existing infrastructure. `--headed` is a 2-line config change. Pixel diff extends the existing png-compare.ts decoder. Clipboard uses page.evaluate. DevTools inspect uses Chrome's `--remote-debugging-port` flag which Playwright passes through via launch args. The heaviest is pixel diff (needs a minimal PNG encoder). DevTools inspect is simpler than expected — no WebSocket proxy needed, just launch Chrome with a debug port and fetch the DevTools URL from `/json`.

## Tasks

### TASK-001: Add `--headed` mode

Add `BROWSE_HEADED` env var and `--headed` CLI flag. When set, launch Chrome with `headless: false` so the browser window is visible. Useful for debugging agent interactions.

**Files to modify:**
- `src/server.ts` — change `headless: true` to `headless: !process.env.BROWSE_HEADED`
- `src/cli.ts` — extract `--headed` flag before command, set `BROWSE_HEADED=1` in env passed to server spawn

**Type:** feature
**Effort:** S

**Acceptance Criteria:**
- [ ] `browse --headed goto https://example.com` opens a visible Chrome window
- [ ] Without `--headed`, Chrome remains headless (no regression)
- [ ] `BROWSE_HEADED=1 browse goto https://example.com` also works via env var

**Agent:** general-purpose

**Priority:** P0

### TASK-002: Add PNG encoder + diff image generator to png-compare.ts

Extend `src/png-compare.ts` with:
1. `encodePNG(img: DecodedImage): Buffer` — minimal PNG encoder (IHDR + unfiltered IDAT + IEND). Uses `zlib.deflateSync` for compression.
2. `generateDiffImage(base: DecodedImage, curr: DecodedImage, colorThreshold: number): Buffer` — creates a diff image where unchanged pixels are dimmed and changed pixels are highlighted in red.
3. Update `CompareResult` interface to include `diffImage?: Buffer`.
4. Update `compareScreenshots()` to return `diffImage` when images differ.

**Files to modify:** `src/png-compare.ts`

**Type:** feature
**Effort:** M

**Acceptance Criteria:**
- [ ] `encodePNG(decodePNG(buffer))` roundtrips — decoded result matches original pixels
- [ ] `generateDiffImage()` returns a valid PNG with red pixels where images differ
- [ ] Identical images return `diffImage: undefined` (no diff generated)
- [ ] Non-square images (different dimensions) produce a diff padded to max size

**Agent:** general-purpose

**Priority:** P0

### TASK-004: Add clipboard read/write commands

Add `clipboard` to `READ_COMMANDS` in server.ts. Implement in `src/commands/read.ts`:
- `clipboard` — read clipboard text via `page.evaluate(() => navigator.clipboard.readText())`
- `clipboard write <text>` — write text via `page.evaluate(t => navigator.clipboard.writeText(t), text)`

Need to grant clipboard permissions via `context.grantPermissions(['clipboard-read', 'clipboard-write'])` — either on context creation or on first clipboard command.

**Files to modify:**
- `src/commands/read.ts` — add clipboard case
- `src/server.ts` — add `'clipboard'` to READ_COMMANDS

**Type:** feature
**Effort:** S

**Acceptance Criteria:**
- [ ] `browse clipboard write "hello"` then `browse clipboard` returns "hello"
- [ ] `browse clipboard` on a fresh page returns empty string or "(empty clipboard)"
- [ ] Clipboard state persists across commands within the same session

**Agent:** general-purpose

**Priority:** P1

### TASK-005: Add `browse inspect` command for DevTools live debug

Add Chrome's `--remote-debugging-port=0` to launch args (auto-assigns port). Add `inspect` as a meta command that fetches `http://127.0.0.1:{debugPort}/json` to get the active page's `devtoolsFrontendUrl` and returns it.

**Implementation:**
1. In `src/server.ts`, add `'--remote-debugging-port=0'` to Chrome launch args
2. After launch, fetch `http://127.0.0.1:{port}/json/version` to discover the debug port (Chrome prints it to stderr on startup, or we can parse the CDP endpoint)
3. In `src/commands/meta.ts`, add `inspect` case that fetches `/json`, finds the active page's target, returns `devtoolsFrontendUrl`
4. User opens the URL in their browser to see live DevTools

**Files to modify:**
- `src/server.ts` — add debug port launch arg, store debug port
- `src/commands/meta.ts` — add `inspect` case
- `src/server.ts` — add `'inspect'` to META_COMMANDS

**Type:** feature
**Effort:** M

**Acceptance Criteria:**
- [ ] `browse inspect` returns a `devtoolsFrontendUrl` that can be opened in Chrome
- [ ] The DevTools URL shows the correct page (the active tab's page)
- [ ] Works alongside normal browse commands (non-blocking)
- [ ] Without `--headed`, inspect still works (headless Chrome still exposes debug port)

**Agent:** general-purpose

**Priority:** P1

## Failure Modes

| Risk | Affected Tasks | Mitigation |
|------|---------------|------------|
| PNG encoder produces invalid PNG | TASK-002 | Roundtrip test: encode(decode(x)) must match original pixels |
| Clipboard permissions denied in headless | TASK-004 | Grant permissions via context.grantPermissions() before read/write |
| Debug port conflicts with other Chrome instances | TASK-005 | Use port 0 (auto-assign) — Chrome picks a free port |
| DevTools URL not accessible from user's browser | TASK-005 | URL uses 127.0.0.1 — works on localhost. Remote use needs docs. |
| --headed mode breaks in CI/containers | TASK-001 | Document: headed mode requires a display. CI should use headless (default). |
