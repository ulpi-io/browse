## Failure Modes

| Risk | Affected Tasks | Mitigation |
|------|---------------|------------|
| Screenshot pixel differences across environments (anti-aliasing, fonts) | TASK-003 | Use high threshold (1%) for same-page comparison; compare same page to itself, not to a stored baseline |
| State test leaks temp files | TASK-005 | Use unique `/tmp/browse-test-state-{timestamp}` dirs with afterAll cleanup |
| Auth login form submission causes page navigation away from test server | TASK-006 | login.html form has no action — submission stays on same domain; verify via page.url() |
| Download test creates leftover files | TASK-007 | Explicit afterAll cleanup of `/tmp/browse-test-dl.txt` |
| Route tests interfere with other tests via lingering routes | TASK-008 | Call `route clear` in afterAll; use dedicated BrowserManager instance |
| Parallel test files sharing BrowserManager state | All test tasks | Each test file creates its own BrowserManager instance (existing pattern) |
